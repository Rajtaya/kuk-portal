The frontend (`frontend-production-9521.up.railway.app`) was sending API requests directly to the backend (`backend-production-7615.up.railway.app`), making `auth_token` a third-party cookie.

**Fix (4 files):**

| File | Change |
|------|--------|
| `frontend/src/lib/api.ts` | `API_BASE` changed from `process.env.NEXT_PUBLIC_API_URL \|\| '/api'` ‚Üí `'/api'` (always same-origin) |
| `frontend/next.config.ts` | Rewrites always active: `source: '/api/:path*'` ‚Üí `destination: ${BACKEND_URL}/api/:path*` |
| `frontend/.env.production` | Replaced `NEXT_PUBLIC_API_URL=...` with `BACKEND_URL=https://backend-production-7615.up.railway.app` (server-side only, not baked into client bundle) |
| `backend/src/auth/auth.controller.ts` | Cookie `sameSite` changed from `'none'` ‚Üí `'lax'`, `secure` changed from `true` ‚Üí `process.env.NODE_ENV === 'production'` |

## Key Architecture Notes

- **Monorepo:** `backend/` (NestJS 11) + `frontend/` (Next.js 15.5, React 19)
- **Auth:** JWT + RBAC ‚Äî 3 roles (SUPER_ADMIN, STATE_USER, UNIVERSITY_ADMIN)
- **Auth cookies:** First-party via Next.js rewrite proxy (`/api/*` ‚Üí backend). Cookie is `sameSite: lax`, `httpOnly`, `secure` in prod.
- **DB:** PostgreSQL ‚Äî local `uems_local` (`postgresql://aarya@localhost:5432/uems_local`), Railway prod
- **Prisma pinned to 6.19.3** ‚Äî always use `./node_modules/.bin/prisma`, never bare `npx prisma`
- **Filled numbers rule:** Total Filled = Budgeted Filled + SFS Filled (contractual excluded from sanctioned posts page, shown on employees page only)
- **Controls placement:** Dashboard scope bar has ThemeToggle/DarkModeToggle/SignOut; sidebar hides them on `/dashboard` via `isDashboard` check. All other pages show them in sidebar.
- **API routing:** Frontend uses relative `/api` path. `next.config.ts` rewrites to `BACKEND_URL` env var. `NEXT_PUBLIC_API_URL` is no longer used.
