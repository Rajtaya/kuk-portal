### ⚠️ Upload 400 bug — openpyxl zip order vs `validateFileSignature` (worth a real fix)
The first upload failed with **HTTP 400 "not a valid Excel workbook"**. Root cause: **openpyxl writes the `.xlsx` zip with `[Content_Types].xml` and `_rels/.rels` LAST**, but `validateFileSignature` (`backend/src/common/file-validation.util.ts`) only scans the **first 4096 bytes** for those markers → it rejects any validly-formed xlsx whose zip orders them late (i.e. **anything generated by openpyxl/Python**). exceljs (what the app uses) front-loads them, so the app's own template passes.
- **Workaround applied:** the generator now repackages the zip so `[Content_Types].xml` is the **first** entry (harmless — Excel/exceljs read the central directory, not byte order). Verified it then passes the exact backend check and still opens as a clean 321-row workbook.
- **Suggested hardening:** make `validateFileSignature` scan the whole buffer / central directory instead of the first 4 KB, so legit openpyxl files aren't rejected. *(2026-06-25 session 2: the **importer** now maps category/type aliases so raw values no longer silently default to UR — but this `validateFileSignature` whole-buffer scan is **still pending**. ## Key Architecture Notes

- **Monorepo:** `backend/` (NestJS 11) + `frontend/` (Next.js 15.5, React 19)
- **Auth:** JWT + RBAC — 3 roles (SUPER_ADMIN, STATE_USER, UNIVERSITY_ADMIN). **As of 2026-06-25, `SUPER_ADMIN` is a full superuser — `RolesGuard` bypasses every `@Roles` check, so it has write access to all universities' data.** `UNIVERSITY_ADMIN` is scoped to its own university; `STATE_USER` is read-only.
- **Auth cookies:** First-party via Next.js rewrite proxy (`/api/*` → backend). Cookie is `sameSite: lax`, `httpOnly`, `secure` in prod.
- **DB:** PostgreSQL — local `uems_local` (`postgresql://aarya@localhost:5432/uems_local`), Railway prod
- **Prisma pinned to 6.19.3** — always use `./node_modules/.bin/prisma`, never bare `npx prisma`
- **Filled numbers rule:** Total Filled = Budgeted Filled + SFS Filled (contractual excluded from sanctioned posts page, shown on employees page only)
- **Controls placement:** Dashboard scope bar has ThemeToggle/DarkModeToggle/SignOut; sidebar hides them on `/dashboard` via `isDashboard` check. All other pages show them in sidebar.
- **API routing:** Frontend uses relative `/api` path. `next.config.ts` rewrites to `BACKEND_URL` env var. `NEXT_PUBLIC_API_URL` is no longer used.
