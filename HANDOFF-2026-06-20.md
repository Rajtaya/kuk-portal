### Cross-origin auth fix details

**Fix (4 files):**

| File | Change |
|------|--------|

| `frontend/next.config.ts` | Rewrites always active: `source: '/api/:path*'` â†’ `destination: ${BACKEND_URL}/api/:path*` |

## Key Architecture Notes

- **Monorepo:** `backend/` (NestJS 11) + `frontend/` (Next.js 15.5, React 19)
- **Auth:** JWT + RBAC â€” 3 roles (SUPER_ADMIN, STATE_USER, UNIVERSITY_ADMIN)
- **Auth cookies:** First-party via Next.js rewrite proxy (`/api/*` â†’ backend). Cookie is `sameSite: lax`, `httpOnly`, `secure` in prod.
- **DB:** PostgreSQL â€” local `uems_local` (`postgresql://aarya@localhost:5432/uems_local`), Railway prod
- **Prisma pinned to 6.19.3** â€” always use `./node_modules/.bin/prisma`, never bare `npx prisma`
- **Filled numbers rule:** Total Filled = Budgeted Filled + SFS Filled (contractual excluded from sanctioned posts page, shown on employees page only)
- **Controls placement:** Dashboard scope bar has ThemeToggle/DarkModeToggle/SignOut; sidebar hides them on `/dashboard` via `isDashboard` check. All other pages show them in sidebar.
- **API routing:** Frontend uses relative `/api` path. `next.config.ts` rewrites to `BACKEND_URL` env var. `NEXT_PUBLIC_API_URL` is no longer used.
