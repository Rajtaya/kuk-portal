## Key Gotchas
- **Prisma v6 pinned:** Global Prisma is v7, project pins v6. Always use `./node_modules/.bin/prisma`, never bare `npx prisma`. Use `prisma db push` (not `prisma migrate dev`) due to DB drift
- **NestJS build path:** Output goes to `dist/src/main.js`, not `dist/main.js`
- **Railway monorepo:** Uses `RAILWAY_DOCKERFILE_PATH=backend/Dockerfile` env var. Dockerfiles use `COPY backend/` (project root context)
- **Next.js cache:** Sometimes needs `.next/` deleted + server restart to pick up changes
- **Frontend env for Railway:** `NEXT_PUBLIC_API_URL` must be set as Docker build arg (baked at build time, not runtime). Also set in `.env.production`
- **Deploy:** `git push` to `main` auto-deploys both services on Railway. Push goes to both `kuk-portal` and `UEMS` repos automatically
- **Railway CDN cache:** Root layout has `export const dynamic = 'force-dynamic'` + `Cache-Control: no-cache` in next.config.ts. Don't remove these
- **Dockerfile must copy public/:** standalone Next.js build does NOT auto-include `public/`. Dockerfile has `COPY --from=builder /app/public ./public`
- **Frontend dev:** port 3001, backend port 4000. Backend start: `npm run dev` (NOT `npm run start:dev`)
- **Login automation:** use `admin@he.haryana.gov.in` / `admin123` for SUPER_ADMIN, `admin@mdu.ac.in` / `admin123` for UNIVERSITY_ADMIN
- **DB sync:** `pg_dump` local → `pg_restore` to Railway. Always use `--clean --if-exists --no-owner --no-acl`
- **Never add Co-Authored-By** lines to git commits (user preference)
